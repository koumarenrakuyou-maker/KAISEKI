## B. J-LIWC2015 の限界点（50項目）

| 優先度 | # | 限界点 |
|:---:|:---:|:---|
| ★★★ | 1 | **辞書カバレッジの低さ**：感情描写での辞書カバレッジが約60%と低い |
| ★★★ | 2 | **形態素解析依存**：MeCabの分割精度に結果が大きく左右される |
| ★★★ | 3 | **複合語問題**：形態素解析器が辞書の1エントリを分割してしまう |
| ★★★ | 4 | **文脈非依存性**：LIWCと同様に文脈・皮肉・比喩を理解不能 |
| ★★★ | 5 | **カテゴリ数の制限**：69カテゴリとLIWC-22の117に比べ大幅に少ない |
| ★★★ | 6 | **敬語体系の未対応**：丁寧語・尊敬語・謙譲語の心理的分析が欠如 |
| ★★★ | 7 | **オノマトペの不十分な対応**：日本語特有の擬音語・擬態語の分類が不完全 |
| ★★★ | 8 | **LIWC-22非対応**：LIWC-22の新カテゴリ・新機能に未更新 |
| ★★★ | 9 | **否定表現の処理不能**：「嬉しくない」の「嬉しい」を肯定感情として計上 |
| ★★★ | 10 | **新語・ネットスラング非対応**：SNS言語やギャル語等の現代表現に非対応 |
| ★★☆ | 11 | GUI不在でR操作の知識が必須 |
| ★★☆ | 12 | Narrative Arc等のLIWC-22新機能が未実装 |
| ★★☆ | 13 | LSM（Language Style Matching）機能の未移植 |
| ★★☆ | 14 | 時制カテゴリの欠如（助動詞ベースの時制分析が限定的） |
| ★★☆ | 15 | 漢字の読み方による意味の違いに非対応（例：「生」の多義性） |
| ★★☆ | 16 | 方言対応の欠如（関西弁、東北弁等） |
| ★★☆ | 17 | 文末表現（〜ね、〜よ、〜かな）の心理的意味分析が不足 |
| ★★☆ | 18 | 受動態・使役態の心理的ニュアンスの区別が不可 |
| ★★☆ | 19 | カタカナ外来語の辞書カバレッジが限定的 |
| ★★☆ | 20 | 形態素解析辞書の選択（IPAdic vs UniDic等）による結果の不一致 |
| ★★☆ | 21 | 絵文字・顔文字の分析に非対応 |
| ★★☆ | 22 | 要約変数の算出が未実装 |
| ★★☆ | 23 | テキスト長による結果の不安定性（短文での精度低下） |
| ★★☆ | 24 | 日本語特有の省略表現の処理が困難 |
| ★★☆ | 25 | 句読点パターンの分析機能が欠如 |
| ★☆☆ | 26 | 主語省略文の処理が困難（日本語は主語省略が頻出） |
| ★☆☆ | 27 | 終助詞の心理的意味のカテゴリ化が不十分 |
| ★☆☆ | 28 | 待遇表現（ウチ・ソト関係）の分析が欠如 |
| ★☆☆ | 29 | 日本語のモダリティ表現の分類が不十分 |
| ★☆☆ | 30 | テキスト可視化機能の欠如 |
| ★☆☆ | 31 | 共起分析機能の欠如 |
| ★☆☆ | 32 | 係り受け解析との連携が不在 |
| ★☆☆ | 33 | 漢字の書き分け（同音異義語）による誤分類 |
| ★☆☆ | 34 | インストールの複雑さ（3種類の辞書が必要） |
| ★☆☆ | 35 | Windowsでの非ASCII文字パス問題 |
| ★☆☆ | 36 | Python非対応（Rのみ） |
| ★☆☆ | 37 | リアルタイム分析が困難 |
| ★☆☆ | 38 | 日本語口語表現のカバレッジ不足 |
| ★☆☆ | 39 | 古語・文語体への対応が欠如 |
| ★☆☆ | 40 | 日本語のアスペクト表現（〜ている等）の分析が限定的 |
| ★☆☆ | 41 | 感情強度の測定が不可 |
| ★☆☆ | 42 | 助数詞の扱いが不明確 |
| ★☆☆ | 43 | テキストジャンルごとの基準値が未整備 |
| ★☆☆ | 44 | 並列処理に非対応（大量データ処理が遅い） |
| ★☆☆ | 45 | クラウドAPI提供がなくローカル実行のみ |
| ★☆☆ | 46 | ドキュメントが一部英語のみ |
| ★☆☆ | 47 | エラーハンドリングが限定的 |
| ★☆☆ | 48 | MeCab以外の形態素解析器対応が限定的 |
| ★☆☆ | 49 | 分析結果の統計的有意性テストが未統合 |
| ★☆☆ | 50 | バージョン間の後方互換性保証が不明確 |
