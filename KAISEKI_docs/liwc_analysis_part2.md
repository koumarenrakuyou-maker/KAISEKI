## B. LIWC-22 の限界点（50項目）

| 優先度 | # | 限界点 |
|:---:|:---:|:---|
| ★★★ | 1 | **文脈非依存性**：皮肉、アイロニー、比喩を理解できない |
| ★★★ | 2 | **否定処理不能**：「happy」と「not happy」を区別できない |
| ★★★ | 3 | **多義語問題**：同じ単語の異なる意味を区別不能 |
| ★★★ | 4 | **ドメイン特異語の欠如**：専門用語・技術用語が辞書に不在 |
| ★★★ | 5 | **新語・スラング対応不足**：ネット言語やSNS特有の表現に非対応 |
| ★★★ | 6 | **要約変数のブラックボックス化**：算出アルゴリズムが非公開 |
| ★★★ | 7 | **LLMに比べ予測精度が劣る**：BERT/GPT等の文脈モデルに精度で劣後 |
| ★★★ | 8 | **静的辞書方式**：言語の動的変化に追従できない |
| ★★★ | 9 | **短文テキストでの不安定性**：250語未満のテキストでは結果が不安定 |
| ★★★ | 10 | **文全体の意味を捉えられない**：単語単位のカウントのみで文の構造を無視 |
| ★★☆ | 11 | 商用ライセンスが必須で高コスト |
| ★★☆ | 12 | 辞書の定期更新に依存（言語変化に遅れる） |
| ★★☆ | 13 | 文化的ニュアンスの捕捉が困難 |
| ★★☆ | 14 | 意図的な言語操作（欺瞞）に脆弱 |
| ★★☆ | 15 | 単語の重要度重み付けがない（TF-IDF等の概念なし） |
| ★★☆ | 16 | 語順・構文情報を完全に無視 |
| ★★☆ | 17 | 共起関係の分析が不可 |
| ★★☆ | 18 | 暗黙の意味（含意）を検出不能 |
| ★★☆ | 19 | 階層的カテゴリ構造の柔軟性が限定的 |
| ★★☆ | 20 | 感情の強度（度合い）を測定できない |
| ★★☆ | 21 | テキスト間の意味的類似度を直接計算できない |
| ★★☆ | 22 | 話者の意図・目的の推定が不可 |
| ★★☆ | 23 | マルチモーダル分析（音声、画像との統合）に非対応 |
| ★★☆ | 24 | 対話構造（ターンテイキング等）の分析が限定的 |
| ★★☆ | 25 | ナラティブ分析に最低250語が必要 |
| ★☆☆ | 26 | 句読点・記号の心理的意味を分析できない |
| ★☆☆ | 27 | 繰り返し表現の心理的効果を捕捉できない |
| ★☆☆ | 28 | 文体レジスター（フォーマル/カジュアル）の自動判定が困難 |
| ★☆☆ | 29 | 引用・間接話法の処理が不十分 |
| ★☆☆ | 30 | 条件文・仮定法の意味分析が不可 |
| ★☆☆ | 31 | ヘッジ表現の統一的な処理メカニズムが欠如 |
| ★☆☆ | 32 | テキスト内の話題転換を検出できない |
| ★☆☆ | 33 | メタファー・慣用句の意味解釈が不可 |
| ★☆☆ | 34 | テキストの読みやすさ指標が未統合 |
| ★☆☆ | 35 | 論理的一貫性の評価機能がない |
| ★☆☆ | 36 | エンティティ認識（人名、地名等）が不可 |
| ★☆☆ | 37 | 感情遷移パターンの自動検出が限定的 |
| ★☆☆ | 38 | 複数文にまたがる指示表現の解決が不可 |
| ★☆☆ | 39 | 書き手の年齢・性別推定精度が限定的 |
| ★☆☆ | 40 | リアルタイム分析（ストリーミング処理）に非対応 |
| ★☆☆ | 41 | 辞書に含まれない語（OOV）の処理方針が不明確 |
| ★☆☆ | 42 | 形態素変化の豊富な言語での精度低下 |
| ★☆☆ | 43 | 絵文字・顔文字の包括的対応が不十分 |
| ★☆☆ | 44 | コードスイッチング（多言語混在テキスト）に非対応 |
| ★☆☆ | 45 | 受動態vs能動態の心理的意味を区別できない |
| ★☆☆ | 46 | 論証構造（主張-根拠-結論）の分析が不可 |
| ★☆☆ | 47 | ユーモア・ジョークの検出が不可 |
| ★☆☆ | 48 | テキストの信頼性・事実性の評価機能がない |
| ★☆☆ | 49 | 社会的ネットワーク構造の推定が不可 |
| ★☆☆ | 50 | APIの機能が限定的で大規模システム統合が困難 |
